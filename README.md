# Incident Response Plan — NovaPay Fintech Ltd.

A formal Incident Response Plan for a fictional fintech startup, built as a portfolio project to demonstrate SOC and GRC analyst skills.

## What's Inside

- Incident Response Team structure with roles and escalation paths
- Incident classification matrix (P1-P4) mapped to Wazuh SIEM alert levels
- 6-phase IR process aligned to NIST SP 800-61
- 3 detailed playbooks:
  - Data Breach (P1 Critical)
  - Ransomware Attack (P1 Critical)
  - Insider Threat / Unauthorized Access (P2 High)
- GDPR Article 33 — 72-hour breach notification procedure
- Internal and customer communication templates
- Post-incident report template
- Wazuh SIEM detection rule mapping to IR actions
- Plan maintenance and testing schedule

## Frameworks Applied

- NIST SP 800-61 (Incident Response)
- ISO 27001:2022
- GDPR Articles 33 & 34
